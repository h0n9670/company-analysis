# Week 1 — 기업 분석 프레임워크

AI 심화 PM 부트캠프 8기 W1D1·W1D2 강의자료에서 기업/시장 분석에 필요한 내용만 추출·정리한 문서.

- **W1D1** = 고객·문제 분석 마인드셋 (보조)
- **W1D2** = 기업/시장 분석 프레임워크 (핵심)

---

## 0. 왜 기업/시장 분석인가 (W1D2 5–6)

- CB Insights: 스타트업 실패 원인 **42%가 "시장 니즈 부재"** — 제품 자체보다 **잘못된 시장 진입**이 더 치명적.
- 시장 분석 = 진입하려는 시장을 체계적으로 이해하는 과정:
  1. 고객 니즈·행동
  2. 경쟁 구도·진입장벽
  3. 산업 동향·기술·규제
  4. 시장 규모·성장성·수익성
- → **Product-Market Fit** 을 찾기 위한 필수 단계.

---

## 1. 시장 규모 — TAM / SAM / SOM (W1D2 7–11)

| 구분 | 정의 | 계산 |
|---|---|---|
| **TAM** (Total Addressable Market) | 제품 카테고리 전체 잠재시장 (이론상 최대) | 총 잠재고객 수 × 연간 고객당 수익 |
| **SAM** (Serviceable Available Market) | 현재 BM·제품·지역으로 실제 서비스 가능한 시장 | 접근 가능 세그먼트 고객 수 × 현재 가격 |
| **SOM** (Serviceable Obtainable Market) | 단·중기적으로 현실적으로 획득 가능한 점유율 | 영업역량 × 제품경쟁력 × 침투율 |

**목적**: 현실적 기대치 설정, 투자 결정 근거, 집중할 세그먼트 명확화, **SOM → SAM → TAM 확장 성장 스토리** 구축.

---

## 2. 산업 구조 — 포터의 5 Forces (W1D2 12)

1. **기존 경쟁사 간 경쟁** — 경쟁자 수·규모·차별화 정도
2. **신규 진입자 위협** — 진입장벽, 초기 투자·규제
3. **대체재 위협** — 니즈를 다른 방식으로 충족하는 제품
4. **공급자 협상력** — 핵심 리소스 공급자 영향력
5. **구매자 협상력** — 가격 민감도, 전환 비용, 선택권

---

## 3. 경쟁사 분석 (W1D2 13–17)

**유형화**
- 직접 경쟁사 — 동일 니즈를 유사 방식으로 해결 (넷플릭스 vs 디즈니+)
- 간접 경쟁사 — 동일 니즈를 다른 방식으로 해결 (넷플릭스 vs 유튜브)
- 잠재 경쟁사 — 진입 가능성 있는 기업 (넷플릭스 vs 메타 콘텐츠 사업)

**분석 방법 5가지**
1. 경쟁사 제품 직접 사용 (전체 경험·강약점 파악)
2. 경쟁사 고객 인터뷰 (불만족·전환 조건)
3. 웹사이트·마케팅 자료 분석 (가치제안·타겟·메시징)
4. 리뷰·SNS 분석 (감성분석·반복 문제 패턴)
5. 업데이트·로드맵 모니터링 (전략 방향 예측)

**약점 분석** → 제품 기능 / 비즈니스 모델 제약 / 타겟 세그먼트 한계 / 기술·자원 한계 → **약점 발견이 곧 차별화 기회**

> 사례: Zoom이 WebEx·Skype·GoToMeeting의 복잡한 설치·불안정성·호환성 약점을 **단순성·신뢰성·성능**으로 공략.

---

## 4. 차별화 5가지 유형 (W1D2 18–23)

1. **기능적 차별화** — 핵심 문제를 10배 더 잘 해결 (다이슨 사이클론, 아이폰 멀티터치). *단, 단순 기능은 모방 쉬움·기능 과잉 위험*
2. **사용자 경험(UX) 차별화** — 전 과정의 총체적 경험 (애플 생태계, 에어비앤비 예약-체크인-숙박 여정)
3. **비즈니스 모델 차별화** — 가치 전달·수익화 방식 혁신 (Dollar Shave Club 구독, 애플 앱스토어)
4. **브랜드·의미 차별화** — 정체성·가치관 공유 (파타고니아 환경, 나이키 Just Do It)
5. **생태계·네트워크 차별화** — 연결된 제품·서비스·커뮤니티 통합 (애플 HW-SW-서비스, 아마존)

**보조 관점**
- **Zero to One**: 수직적 진보, 독점 시장, **10배 법칙**, 비밀(Secret) 발견 (페이팔·스페이스X·에어비앤비)
- **Uncopyable**: 제품 × 경험 × 연결의 복합 시스템으로 모방 장벽 구축 (애플·스타벅스 '제3의 공간'·테슬라)
- **파괴적 혁신(Clayton Christensen)**: 로우엔드 파괴, 신시장 파괴, 과잉서비스·비소비자 타겟팅 (넷플릭스·저가항공·미니밀스)

## 5. 디커플링 (Decoupling) (W1D2 27–41)

> 핵심: **기술 혁신이 아닌 비즈니스 모델 혁신이 성장의 진짜 동력**
> (디지털 스타트업 20곳 중 18곳에서 특허는 수익의 *원인이 아닌 결과*)

**고객 가치 사슬(CVC)**: 탐색 → 평가 → 선택 → 구매 → 사용 → 처분

**활동 3유형**
1. 가치 창출 활동 — 고객이 즐기고 혜택받는 것 (음악 감상, 제품 사용)
2. 가치 부과 활동 — 대가 지불 (돈·시간·노력: 구매, 이동, 결제)
3. 가치 잠식 활동 — 싫지만 감수 (광고, 복잡한 가입, 대기)

**디커플링 3유형**
- 가치 창출 분리 → **Dollar Shave Club** (핵심 가치만 남기고 기술·유통·셀럽 마케팅 제거 → $1 면도날, 유니레버 $1B 인수)
- 가치 포착 분리 → **Best Buy** (쇼루밍을 제조사 임대·Geek Squad 서비스·멤버십 수익으로 전환 → 신규 수익이 전체의 30%)
- 가치 잠식 분리 → **Uber** (이동 외 모든 마찰 제거: 즉시 호출·실시간 위치·자동 결제·디지털 영수증)

> Best Buy 전환점: "우리는 삼성을 위한 무료 쇼룸을 운영하고 있었다" → Store-within-a-Store 모델 탄생

## 6. 세그멘테이션 (W1D2 42–57)

- **정의**: 공통 특성 그룹으로 분류 — **추측(Guessing)이 아니라 분석(Analyzing)**, 실제 행동 데이터 기반이어야 함
- **종류**: 인구통계 / 지리 / 심리(가치관·라이프스타일) / **행동 기반**(구매·사용 패턴·충성도)

**기술 수용 주기 5단계**

| 단계 | 비율 | 특징 |
|---|---|---|
| Innovator | 2.5% | 호기심·리스크 감수, 최초 도입 |
| Early Adopter | 13.5% | 트렌드 리더·인플루언서, 혁신성 우선 |
| Early Majority | 34% | 검증 후 채택, 실용·신중 |
| Late Majority | 34% | 대중화 후 채택, 가격 민감·보수적 |
| Laggards | 16% | 최후 도입, 전통적 |

- **캐즘(Chasm)**: Early Adopter(기술 자체에 관심) ↔ Early Majority(실질 가치 원함) 사이 간극. 못 넘으면 주류 시장 확산 실패로 제품 소멸.
- ⚠️ **세그멘테이션은 '해석의 도구'이지 '추측의 도구'가 아니다** — 데이터 없는 분류는 추측일 뿐.
  - (X) "이런 고객들은 아마 저런 행동을 할 것이다"
  - (O) "우리 고객군이 이런 행동을 한 이유를 분석하니 저런 성향이었다"

## 7. 데이터·행동 분석 (W1D2 59–98)

- **정량(무엇·얼마나) + 정성(왜·맥락) 통합**. 예: "70% 결제 이탈"(정량) + "복잡·불신"(정성) → "결제 단순화·신뢰 신호 추가"(통합 인사이트)
- **핵심 고객 행동**: 활성도(DAU/WAU/MAU), 리텐션·습관 형성, 핵심 기능 경험, 추천·네트워크 효과
- **비정상 행동**: 이탈(즉각/점진), 기대와 다른 사용, 오류·오용, 이상치(Anomaly)
- **충성 고객 First 7 Days Rule** — 가입 후 7일 내 행동이 장기 리텐션 결정
  - 페이스북: 10일 내 7명 친구
  - 드롭박스: 1개 폴더에 1개 파일 저장
  - 넷플릭스: 첫 에피소드 70% 시청 → 90% 시즌 완주

**데이터 해석의 함정 6가지**
1. 상관관계 ≠ 인과관계 (숨은 변수·우연)
2. 확증 편향 (믿고 싶은 데이터만)
3. 생존 편향 (성공 사례만, 이탈자 배제)
4. 평균의 오류 (분포·중앙값·백분위 함께 봐야)
5. 과도한 패턴화 (Overfitting)
6. 측정 왜곡

## 8. (보조) 고객·문제 분석 도구 — W1D1 발췌

기업이 '어떤 가치로 어떤 문제를 해결하는가'를 분석할 때 활용.

- **가치 교환 / Output vs Outcome**: 기능 수(Output)가 아닌 고객 변화·문제 해결(Outcome)로 가치 측정. 'Build Trap' 경계.
- **JTBD (Jobs To Be Done)**: 고객은 제품이 아니라 '할 일(Job)'을 위해 제품을 '고용' (밀크셰이크 = 출근길 한 손 아침식사). → 경쟁자 재정의(바나나·베이글·도넛도 경쟁자).
- **5 Why**: 표면적 문제 → 근본 원인. *단, 단일 원인 편향 주의 — 피시본·시스템 사고 병행.*
- **시스템 사고**: 선형적(원인→결과) vs 시스템적(상호작용·피드백 루프). "매출 30% 감소" 같은 복합 문제 진단에 필수.
- **퍼소나 / 고객 여정 맵**: 창작이 아니라 데이터에서 '발견'하는 것. 고객의 핵심 문제 발견을 위한 도구일 뿐.
